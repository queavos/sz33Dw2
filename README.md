# Diseño y Programación Web II -- Repositorio de Desarrollo

Repositorio utilizado para registrar el **avance clase a clase** de los
ejemplos, ejercicios y prácticas desarrolladas durante la cursada de
**Diseño y Programación Web II**.

Repositorio oficial del curso:
git clone  https://github.com/queavos/sz33Dw2.git

El objetivo del repositorio es:

-   Registrar el desarrollo **incremental** de los ejemplos de clase.
-   Permitir que los estudiantes **sigan paso a paso la evolución del
    proyecto**.
-   Mostrar cómo se construye una aplicación web **de forma
    progresiva**.
-   Introducir el uso de **Git y GitHub** en un contexto real.

------------------------------------------------------------------------

# 📅 Método de organización del repositorio

Este repositorio utiliza una convención simple para organizar el trabajo
de cada clase.

Las carpetas se nombran utilizando el formato:

MESDIA

Ejemplo:

0305 0312 0319 0326

Esto significa:

  Carpeta   Clase
  --------- -----------------------
  0305      Clase del 5 de marzo
  0312      Clase del 12 de marzo
  0319      Clase del 19 de marzo
  0326      Clase del 26 de marzo

------------------------------------------------------------------------

# 🔁 Desarrollo incremental

El desarrollo del proyecto se realiza de forma **incremental**.

En cada nueva clase se hace lo siguiente:

1.  Se **duplica la carpeta de la clase anterior**.
2.  Se renombra con la **fecha de la nueva clase**.
3.  Se continúa el desarrollo desde ese punto.

Ejemplo:

0305 0312

La carpeta `0312` es inicialmente una copia de `0305`, y sobre ella se
continúa trabajando.

Esto permite que:

-   cada clase tenga una **foto exacta del estado del proyecto**
-   los estudiantes puedan **revisar cómo evolucionó el código**
-   el aprendizaje sea **progresivo y trazable**

------------------------------------------------------------------------

# 📚 Ejemplo de evolución del proyecto

Ejemplo hipotético:

0305 │ ├── index.html ├── estilos.css └── script.js

En la siguiente clase:

0312 │ ├── index.html ├── estilos.css ├── script.js └── formulario.html

Luego:

0319 │ ├── index.html ├── estilos.css ├── script.js ├── formulario.html
└── procesar.php

De esta forma es posible observar claramente:

-   qué archivos se agregaron
-   qué código cambió
-   cómo se construyó la aplicación paso a paso

# 🔎 Cómo seguir el desarrollo del proyecto

Para entender cómo evoluciona el proyecto:

1.  Abrir la carpeta más antigua.
2.  Revisar su contenido.
3.  Pasar a la carpeta de la siguiente fecha.
4.  Comparar qué cambió.

De esta forma se puede reconstruir **todo el proceso de desarrollo**
realizado durante las clases.
